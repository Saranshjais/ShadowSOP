# Plan 6.1: Environment Separation & CORS

## Objective
Update backend and frontend code to dynamically accept environment variables for production (Render).

## Context
- .gsd/SPEC.md
- backend/main.py
- frontend/src/components/Forge.tsx
- frontend/src/components/Vault.tsx

## Tasks

<task type="auto">
  <name>Update Backend CORS</name>
  <files>backend/main.py</files>
  <action>
    Modify `CORSMiddleware` to include `os.getenv("CORS_ORIGINS", "*")` in the `allow_origins` array.
    Ensure `import os` is added to `main.py`.
  </action>
  <verify>grep -q CORS_ORIGINS backend/main.py && echo 'pass'</verify>
  <done>Backend dynamically handles CORS based on environment.</done>
</task>

<task type="auto">
  <name>Update Frontend API URLs</name>
  <files>frontend/src/components/Forge.tsx, frontend/src/components/Vault.tsx</files>
  <action>

    Replace hardcoded `http://localhost:8000` with `${API_BASE_URL}` in all fetch calls.
  </action>
  <verify>grep -q VITE_API_URL frontend/src/components/Forge.tsx && echo 'pass'</verify>
  <done>Frontend dynamically points to the correct backend API URL.</done>
</task>
